### 1 ¬∑ `Cases.jsx` ‚Äî Add `status` field to the create/edit form

**Goal.** Allow a clerk or admin to set (or override) the case status at registration / edit time. Backend already supports `status` as a writable field (default `pending`).

**Steps**
1. Add `status` key to `emptyForm` ‚Äî default `'pending'`.
2. Add `status` to the initial `form` state destructuring in the `handleSave` payload so it is always sent.
3. Add `status` to the field block inside `openEdit` so existing cases load their current status.
4. Insert a `<select>` for `status` in the modal form, between the Case Type and Case Title fields.
   - Use the existing `STATUSES` array (`['pending','scheduled','heard','adjourned']`).
   - Map to capitalised option labels for consistency with the filter dropdown.
5. Ensure `handleSave` sends `status` in the POST/PUT body without transforming it to/from a number (it is a string).
6. Run lint / typecheck (`npm run build`) to confirm no regressions.

### 2 ¬∑ `Judges.jsx` ‚Äî Verify & tighten the confirm/activate-deactivate flow

**Goal.** The `handleToggleActive` function on line 97 is still functional but uses an incorrect API contract:
- **Deactivate path (active ‚Üí inactive):** sends a `DELETE /judges/:id` which deletes the record entirely ‚Äî a data-loss bug, not a deactivation.
- **Activate path (inactive ‚Üí active):** sends `PUT /judges/:id` with `{ ...judge, is_active: true }` ‚Äî acceptable but fragile because the whole judge object is sent.

**Steps**
1. Change the `confirm` action structure so the active/inactive state is more explicit. Replace `action: 'activate' | 'deactivate'` strings with a `type` / `activate` boolean:
   ```jsx
   setConfirm({ judge: j, action: 'deactivate' })  // keep labels intact
   ```
   ‚Üí keep the existing string shape for now; only fix the **DELETE ‚Üí PUT** bug.
2. In `handleToggleActive`, when `judge.is_active` is `true`, send `PUT` with `{ is_active: false }` instead of `DELETE`.
3. Confirm `ConfirmDialog` is rendered outside any conditional that could unmount it ‚Äî it already is (lines 290‚Äì302).
4. Update the plan file to mark this as "in progress / resolved" when confirmed.

**Diff shape (conceptual)**
```

```
