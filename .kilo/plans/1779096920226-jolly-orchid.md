# Gap Analysis Frontend Implementation Plan

## Overview
Create a new `/gap-analysis` page that analyzes resource allocation gaps in the court scheduling system, showing mismatches between available resources (judges, courtrooms) and workload demands.

## Feature Requirements

### 1. Main Gap Analysis Page (`Frontend/src/pages/GapAnalysis.jsx`)
**Purpose:** Identify and visualize scheduling inefficiencies and resource gaps.

**Data Visualizations:**
- **Judge Workload vs Capacity Matrix** - Compare assigned cases vs judge capacity
- **Courtroom Utilization Heatmap** - Show usage patterns by day/time
- **Case Distribution Gaps** - Identify case types with insufficient judge coverage
- **Time Slot Availability Gaps** - Highlight underutilized/underbooked time slots

**Metrics to Display:**
- Cases per judge capacity ratio (current vs optimal)
- Courtroom occupancy percentages by day
- Unscheduled case backlog by priority
- Judge availability gaps (vacation, limited hours)

### 7. Data Calculations (Backend Logic)

**Judge Gap Calculation:**
```sql
-- Current load vs max_hearings_per_day * working_days
SELECT j.id, j.name, j.max_hearings_per_day,
       COUNT(c.id) as current_load,
       ROUND((COUNT(c.id) * 100.0 / (j.max_hearings_per_day * 5)), 2) as gap_percentage
FROM judges j
LEFT JOIN cases c ON c.assigned_judge_id = j.id AND c.status IN ('pending', 'scheduled')
GROUP BY j.id
```

**Courtroom Gap Calculation:**
```sql
-- Booked hearings vs total possible slots (8 hours/day * 5 days * courtrooms)
SELECT cr.name,
       COUNT(h.id) as booked_slots,
       (8 * 5) as total_slots,
       ROUND(COUNT(h.id) * 100.0 / (8 * 5), 2) as utilization_rate
FROM courtrooms cr
LEFT JOIN hearings h ON h.courtroom_id = cr.id
WHERE h.scheduled_date >= date('now', '-30 days')
GROUP BY cr.id
```
